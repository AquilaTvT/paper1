## 3. 第 4 章：总体设计

### 4.1 系统架构

可引用 `docs/PROJECT_SPEC.md` 中的四层架构：

- 前端层：Vue。
- 业务服务层：Java Spring Boot。
- 推理计算层：Python FastAPI。
- 中间件层：Redis。

论文架构图建议包含：

```text
浏览器前端
   ↓ REST / SSE
Java Spring Boot 业务服务
   ↓ Redis 队列 / 状态 / 流消息
Python FastAPI 推理服务
   ↓
视频预处理 + Video Swin + 双轨 Token 压缩 + MLP Adapter + 摘要生成
```

### 4.2 模块设计

论文第 4 章可按以下模块展开：

- 视频上传模块。
- 任务管理模块。
- Redis 队列模块。
- SSE 流式输出模块。
- 视频预处理模块。
- Video Swin 特征提取模块。
- 双轨 Token 压缩模块。
- MLP Projection Adapter 模块。
- 摘要生成模块。
- 结果展示模块。
- 日志与异常处理模块。

### 4.3 数据流设计

标准数据流：

1. 前端上传视频到 Java 后端。
2. Java 后端保存文件并返回 `videoId`。
3. 前端基于 `videoId` 创建任务。
4. Java 后端生成 `InferenceTask`，写入 Redis 队列。
5. Python worker 消费任务并执行流水线。
6. Python worker 写入状态、流式消息、指标和结果。
7. Java 后端通过 SSE 将消息转发给前端。
8. 前端展示摘要与指标。

## 4. 第 5 章：详细设计

### 5.1 接口详细设计

可引用 `docs/API_CONTRACT.md`，重点说明：

- 上传接口的 multipart 设计。
- 任务创建接口的参数设计。
- 任务查询接口的分页和状态字段。
- SSE 事件类型设计。
- 结果接口中的 TokenMetrics。

### 5.2 数据模型详细设计

可引用 `docs/DATA_MODEL.md`，重点说明：

- `VideoFile` 保存视频元数据。
- `InferenceTask` 管理任务生命周期。
- `InferenceResult` 保存摘要和指标。
- `TaskLog` 支撑测试分析和异常定位。
- `StreamMessage` 支撑流式输出。
- `TokenMetrics` 体现 `196 → 5` 的压缩效果。

### 5.3 算法流程详细设计

论文可按以下流程描述：

1. 视频预处理：视频解码、帧采样、尺寸归一化。
2. Video Swin Transformer 特征提取：获得时空视觉特征。
3. Content Token 与 Context Token 双轨压缩：将单帧 196 个 Patch Token 压缩为 5 个视觉 Token。
4. MLP Projection Adapter：将视觉 Token 映射至文本生成模型语义空间。
5. 摘要生成：结合用户指令生成摘要、关键事件和时间线。

### 5.4 mock mode 与 real mode 详细设计

- mock mode 用于完整演示和论文截图，保证普通电脑可运行。
- real mode 用于保留真实模型接入能力，支持后续实验扩展。
- 两种模式共享同一接口契约和数据模型，降低前后端改动成本。

### 7.1 功能测试用例

| 编号 | 测试项 | 输入 | 期望结果 |
